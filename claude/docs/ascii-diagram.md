# ASCII図の整列

プログラミングフォント前提(CJK=2col, ASCII/box-drawing/→←↓↑=1col)。

## 原則: 座標計算で配置(目視NG)

書く前に列レイアウトを先に決める:
```
- Box1: cols 0-9 (10w)
- Gap:  cols 10-29 (20w, arrow配置)
- Box2: cols 30-45 (16w)
```

各文字を「視覚col幅」で数えてから配置。目分量で揃えると必ずズレる。

## 文字幅ルール

| 種別 | col幅 | 例 |
|------|------|----|
| ASCII半角 | 1 | `A`, `1`, `(`, ` ` |
| 罫線 | 1 | `─│┌┐└┘├┤┬┴┼` |
| 矢印 | 1 | `→←↓↑` |
| 全角(漢字/カナ) | 2 | `商`, `ア`, `、`, `（` |

注意: `→←↓↑`は East Asian Ambiguous でフォントにより2col扱いの場合あり。プログラミングフォントなら通常1col。

## Box幅の整合

枠線・内容・底辺の幅を必ず一致させる:
```
┌──────────────┐   ← 1+14+1 = 16w
│ 商品検索機能 │   ← 1+1+12+1+1 = 16w (商品検索機能=12col)
└──────────────┘   ← 16w
```

NG例: 枠線`┌─────────────┐`(15w)に対し内容`│商品検索機能│`(14w)。

## ギャップ計算

「BoxAの終端col」と「BoxBの開始col」が決まれば、間の空白数は逆算:
```
BoxA(cols 0-9), BoxB(cols 30-45)
→ 間は cols 10-29 (20col)
→ "          ─────────────────→" (10sp + 17─ + → = 28chars) NG、合わない
→ 列計算: 10sp + 19─ + → = 30chars、cols 10-29を埋める ✓
```

## 連結点(┬┼┘)の中心軸固定

垂直線で複数行を繋ぐ場合、全行で同じcolに連結文字を置く:
```
└──────┬───────┘   ← ┬ at col 45
       │            ← │ at col 45
┌──────┴───────┐   ← ┴ at col 45 (or ┬ for branching)
```

## ツリー/バス構造の累積ズレ

複数区間を等間隔で繋ぐとき、ダッシュ数を1個間違えると右に累積ズレ発生:
```
NG: ┌─×21┬─×21┼─×21┬─×21┐ (子Box中心 8,30,52,74,96)
OK: ┌─×20┬─×20┼─×20┬─×20┐ (子Box中心 8,29,50,71,92) ← 子Box位置と一致
```

子要素の中心位置を先に決めて、そこから区間幅(=中心間距離-1)を逆算する。

## ワークフロー

1. 全構成要素の幅をリストアップ(Box, ギャップ, 矢印)
2. col位置を決める(0始まり)
3. 各行で「要素 + 空白」を順に書き、累計colを検算
4. 連結文字(┬┼┘等)は全行で同じcolにあることを確認
5. 書いた後、`Read`で確認し、ズレ箇所のcol位置を特定して修正

## 代替案: Mermaid

複雑な図はMermaidに逃げるのが楽。VSCode/GitHubで正しくレンダリングされ全角/半角問題が無い:
```mermaid
flowchart LR
    A[利用者] -->|商品名| B[商品検索機能]
    B -->|検索結果| A
```

ただしユーザーがプログラミングフォント+ASCII図を好む場合は座標計算で対応する。
