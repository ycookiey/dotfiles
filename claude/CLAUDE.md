# ルール

- 作業開始前に何をするか簡潔に示す(1-2行)
- シェルはGit Bash。Windowsパスは Unix形式で /c/path。Edit/Read は C:\path
- 未完成の実装にはTODOコメント必須
- IMPORTANT: コミットメッセージにAIツール名や Co-Authored-By を入れない
- ハマった問題: まず汎用的な学びを抽出し ~/.claude/docs/ に具体的ファイルで記録
- 不明点や困ったことは積極的にWebSearchで解決する
- 外部サービスのURLを案内時,クエリパラメータを活用しユーザ操作を減らす
- 対処法が複数あり得る問題は、最初に見つけた方法を即実装せず、トレードオフを比較提示してユーザが選んでから実装する
- 指示が曖昧で誤った方向に進むリスクがあれば推測せず質問。候補があればAskUserQuestionでもよい
- 類似ファイル作成はcp/mv + Edit
- 変更を元に戻すにはgit checkout（Editで逆変換しない）

# 自己改善

以下のシグナルを検知したら、学びの永続化を skill `meta-update` で提案する。

- 再作業: 同じ調査/回答/手順を同一会話で2回以上
- 誤誘導: 命名/説明/skill/文書が原因の誤解(自分固有でなく誰が読んでも誤読する時)
- 行動指針: 「そうじゃない」「毎回〜して」等、次回以降に効く規範をユーザが示した
- 陳腐化: 既存のCLAUDE.md/docs/skill記述が実態(コード/構成/設定)と乖離しているのを作業中に発見(実害の有無を問わない)

タイミングは2段階: ①検知した時点では1行告知のみ(例「陳腐化検知: X」、タスクは中断しない) ②`meta-update` の起動と提案はタスク区切りで。ただしタスクが確認待ち等で停止する場合は、その応答内に①の告知を必ず含め取りこぼさない。
